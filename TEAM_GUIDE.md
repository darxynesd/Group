## Швидкий старт

1. **Установка залежностей:**
```bash
pip install -r requirements.txt
```

2. **Запуск сервера:**
```bash
python manage.py runserver
```

3. **Створення суперкористувача:**
```bash
python manage.py createsuperuser
```

4. **Відкрити у браузері:**
http://127.0.0.1:8000/

## Розподіл роботи за командами

### Backend команда:

#### Пріоритет 1: Моделі
Кожен член команди backend може взяти один або декілька додатків та створити моделі:

**accounts:**
- Розширити User модель (Profile)
- Додати ролі: USER, MODERATOR, ADMIN

**forum:**
- Thread (гілка форуму)
- Post (повідомлення)
- Category (категорії)

**gradebook:**
- Subject (предмет)
- Grade (оцінка)
- Student (студент - можливо пов'язати з User)

**events:**
- Event (подія)
- Calendar (можливо окрема модель)

**surveys:**
- Survey (опитування)
- Question (питання)
- Answer (відповідь)
- UserResponse (відповіді користувачів)

**voting:**
- Poll (голосування)
- Option (варіант відповіді)
- Vote (голос користувача)

**announcements:**
- Announcement (оголошення)

**materials:**
- Material (матеріал)
- File/Link типи

**portfolio:**
- Portfolio (портфоліо)
- Project (проект)

**gallery:**
- GalleryItem (елемент галереї)
- Photo/Video типи

#### Пріоритет 2: Форми
Після створення моделей - створити Django форми для валідації даних.

#### Пріоритет 3: Views
Реалізувати Class-Based Views для CRUD операцій.

#### Пріоритет 4: Права доступу
Налаштувати permissions та decorators для ролей.

### Frontend команда:

## Git workflow:

1. Створіть нову гілку для вашого функціоналу:
```bash
git checkout -b feature/назва-функції
```

2. Робіть commits регулярно:
```bash
git add .
git commit -m "Опис змін"
```

3. Push вашої гілки:
```bash
git push origin feature/назва-функції
```

4. Створіть Pull Request для review

## Корисні команди:

```bash
# Створити міграції після змін у models.py
python manage.py makemigrations

# Застосувати міграції
python manage.py migrate

# Створити суперкористувача
python manage.py createsuperuser

# Запустити тести
python manage.py test

# Збірка статичних файлів (для продакшн)
python manage.py collectstatic

# Перевірка проекту на помилки
python manage.py check
```
